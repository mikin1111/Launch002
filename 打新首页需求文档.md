Gate Web端「打新中心 (Launch Hub)」产品需求文档 V2.0 (Web端专属优化版)

1. 需求背景与优化目标

背景：在 V1 版本将移动端 App 界面平移至 Web 端的尝试中，发现纯纵向列表存在屏幕空间利用率低、信息密度不足（“App化”过重）的问题。

优化目标：

提升信息密度：引入 Web 端的横向宽卡片、网格（Grid）和数据表格（Table）布局，减少用户滚屏成本。

丰富决策维度：从各打新子产品的二级页面中，提取核心决策字段（如：倒计时、参与人数、项目简介、质押币种），前置到总览页。

强化转化路径（CTA）：改变以往点击卡片跳转的单一逻辑，直接在每行/每个卡片右侧暴露核心操作按钮（如“立即认购”、“立即质押”、“一键领取”），缩短转化漏斗。

视觉统一：统一全站卡片风格，采用浅色商务、数据化设计，避免单一模块（如 HODLer Airdrop）过于活动化、跳脱整体框架。

2. 页面整体框架与规范

页面位置：顶部导航栏 -> 币圈 -> 打新中心 -> Tab 1: 打新总览。

页面宽度：内容区域最大宽度限制为 1200px，居中显示。

模块排布顺序（自上而下）：Launchpad 首发认购 -> CandyDrop 免费发糖果 -> Launchpool 质押挖矿 -> 积分任务中心 (Alpha + 合约合并) -> HODLer Airdrop 持仓空投。

3. 各模块详细需求说明

3.1 Launchpad (首发认购) - 横向宽卡片布局

设计意图：作为重磅首发，占据首屏核心位置，采用图文左右分栏设计。

数据字段与展示：

左侧（项目信息区）：

状态标签：带呼吸灯动效的“进行中”状态。

代币Logo与名称：如 E / ENA (Ethena)。

项目简介 (新增)：2行截断展示，如“Ethena 是一个基于以太坊的合成美元协议...”。

热度与时间 (新增)：当前参与人数（如 14,592 参与），结束倒计时（如 02天 14:25:30 结束）。

右侧（数据与操作区）：

发行总量：如 6,000,000 ENA。

平台总额度：如 300,000 ENA。

认购价格 (支持多专场展示)：高亮展示双币种定价，如 1 ENA = 20 USDT (USDT专场) 和 1 ENA = 0.25 GT (GT专场)。

进度条 (新增)：展示当前认购进度百分比（如 已认购 65%）。

交互与操作：

“立即认购”按钮：常驻右下角，点击后呼出认购弹窗或跳转详情。

3.2 CandyDrop (免费发糖果) - Web 表格流布局

设计意图：项目较多，采用 Table 列表形式更利于数据对比和扫读。

全局数据：右上角展示该模块“累计空投收益”。

Tab 筛选：全部项目 (All)、合约专区 (Future)、现货专区 (Spot)、邀请奖励 (Invite Friends)。

表头与字段映射：

代币项目：Logo、名称缩写、项目全称。包含 HOT / NEW 标签。

空投总额度：如 6,000,000 ENA。

最高可赢取 (Win Upto)：如 4,500 ENA。

倒计时 (新增)：精确到秒，如 05:22:10。

操作：常驻操作栏。

交互与操作：

列表单行 Hover 态增加背景微灰。

每行对应一个 “参与”按钮。

3.3 Launchpool (质押挖矿) - 强调高回报的网格卡片

设计意图：理财类产品，最核心的是年化收益（APR），采用 1行3列的 Grid 卡片铺开展示。

全局数据：右上角展示该模块“收益账户余额”。

卡片内数据字段：

基本信息：代币 Logo、名称、状态标签（HOT, New Delivery）。

核心利益点：超大字号、绿色高亮的预估年化收益 (APR)，如 1.04% - 302.82%。

规则说明 (新增，灰色底色块)：

奖池总量：如 6,000,000 ENA。

质押币种 (新增)：以 Tag 形式罗列该池支持的币种（如 USDT, GT, BTC）。

挖矿周期 (新增)：如 14 Days。

交互与操作：

整卡 Hover 时上浮（translate-y-1）并增加外阴影。

卡片底部常驻 “立即质押”按钮，宽度 100%。

3.4 积分任务中心 - 双栏对剖面板

设计意图：将原有的 Alpha Points 和 Futures Points 整合为一个模块，左右各占 50%，节约纵向空间。

全局数据：右上角分别展示两种积分的当前余额。

左侧：Alpha 积分兑换：

列表字段：兑换币种、兑换要求（如“需要积分: 1-49”）、可兑换额度（如“0.0001 ENA”）。

右侧：合约积分兑换：

普通代币列表同上。

特殊卡片：合约体验金 (Futures Bonus) 需要特殊视觉高亮（如金色渐变底色，增加闪电 Icon），强化 100 USDT 体验金的吸引力。

交互与操作：每条任务/兑换项右侧提供文字按钮或小按钮样式的**“兑换”入口**。

3.5 HODLer Airdrop (持仓空投) - 统一风格白底卡片

设计意图：去除过度活动化的深色/发光背景，回归交易平台严谨、清晰的白底数据卡片风格，与其他业务模块保持一致的视觉节奏。

顶部核心区：

巨型文案：“持仓即享收益：持有 GT 即可免费获得热门新币空投”。搭配专属的浅紫色基调 Icon。

累计获取空投金额大字号展示。

“一键领取 (Quick Claim)” 按钮：采用品牌紫色常态按钮，去除发光特效。

下半部分（代币明细 Grid）：

1行3列的横向排列，白底+分割线设计。

字段：代币名称、标签、发行量、每 GT 分配比例 (Ratio)。

底部横幅广告：淡紫色背景条，常驻本期活动的预告，如“$22,500 免费空投现已开启 -> 立即参与”。

4. 交互与体验规范

响应式降级：当屏幕宽度 < 1024px 时，所有的左右分栏（Launchpad、积分中心）、1行3列 Grid（Launchpool、HODLer Airdrop 明细），以及 Table 布局，应自动折叠为单列（1列）流式布局，以保证移动端/Pad端不破版。

状态实时性：倒计时区域（CandyDrop、Launchpad）需要在前端做基于本地时钟（并校对服务器时间）的逐秒递减效果。

按钮反馈：

所有 Primary 按钮（立即认购、参与、立即质押）点击后需有 Loading 状态防抖。

“一键领取”点击成功后，需在页面正中弹出“领取成功，资产已发放入现货账户”的 Toast 提示。
